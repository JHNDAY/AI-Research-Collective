English in Tokens:
Fewer Units, Infinite Expression

How large language models compress the English lexicon into a smaller vocabulary of subword units — and use probability to generate fluent text, one token at a time.

600K Words in the OED
170K Active English words
~100K Tokens in a modern LLM
1 Token predicted per cycle

The Compression Problem

English has roughly 170,000 active words and over 600,000 historical entries in the Oxford English Dictionary. An LLM tokenizer — the component that converts text to numbers — uses only around 100,000 token types in its fixed vocabulary. Yet it can represent every English word that has ever existed, including brand-new ones.

How? Through subword decomposition. Instead of assigning one integer to every word, the tokenizer breaks language into frequently-occurring fragments — morphemes, syllables, common character clusters — and composes words from them.

Tokens Are Not Words

Most common words map 1-to-1 with a token. Rarer, longer, or newer words are split into pieces. Below is a live example of how a sentence is tokenized:

INPUT SENTENCE → TOKEN SEQUENCE
"The " The → 1 token (very common)
"neurodivergent " neurodivergent → 3 tokens (rare compound)
"transformer " transformer → 2 tokens (morpheme split)
"unbelievable" unbelievable → 4 tokens (prefix + root + suffix)
"ChatGPT" ChatGPT → 3 tokens (proper noun, post-training)
green = 1-to-1 with word  |  blue = subword fragment  |  red = rare / proper noun split

Fewer Tokens Than Words: The Comparison

Scope Word Count Approximate Tokens Needed Ratio
Top 10,000 common words 10,000 ~12,000 1.2×
Educated adult vocabulary ~30,000 ~45,000–60,000 1.5–2×
Full active English lexicon ~170,000 ~250,000–400,000 1.5–2.3×
Complete OED ~600,000 ~1–2 million 1.7–3×
LLM tokenizer vocabulary — ~100,000 token types Covers everything

The paradox is elegant: 100,000 token types can represent 600,000+ words because words are constructed compositionally. It mirrors the way 26 letters can spell any English word, and how 12 musical notes can produce any melody.

Key insight A tokenizer vocabulary is not a dictionary. It is a generative basis — a set of primitives from which all words can be built. Fewer units, but unlimited expressibility.

Are Any English Words "Missing"?

In principle, no English word is unrepresentable. The tokenizer includes every character in Unicode, so any word — however obscure — can always be spelled out character by character as a last resort. In practice, very rare or newly coined words are simply tokenized less efficiently:

Word TypeLikely TokenizationCost
High-frequency words (the, run, cat)Single token1 token
Common compounds (bookshelf)2 tokens2 tokens
Technical terms (hepatosplenomegaly)4–7 subword fragments4–7 tokens
Post-training proper nouns (Anthropic)2–3 fragments2–3 tokens
New slang (rizz, delulu)Character-level fallback3–6 tokens
Emoji / symbols1–3 tokens each1–3 tokens

The practical consequence: rare words are expensive. They consume more context-window space, appeared less often in training, and are somewhat harder for the model to reason about. This is why LLMs can occasionally stumble on unusual proper nouns or highly technical vocabulary.

The Generative Process: One Token at a Time

When an LLM generates text, it does not write a sentence. It makes a single probabilistic choice — "what token comes next?" — and repeats that choice hundreds of times until the response is complete.

Step 1
All prior tokens converted to integer IDs
Step 2
Each ID looked up in embedding table → vector
Step 3
Vectors pass through transformer layers (attention + FFN)
Step 4
Final layer outputs one logit score per vocabulary token
Step 5
Softmax converts logits to a probability distribution
Step 6
One token sampled → appended → repeat from Step 1

The Probability Distribution

At each step, the model assigns a probability to every token in its vocabulary — but the distribution is extremely skewed. Almost all the probability mass concentrates on a handful of plausible continuations, while tens of thousands of tokens receive near-zero probability.

Temperature is a parameter that artificially flattens or sharpens this distribution at inference time.

Putting It All Together

The architecture is beautifully recursive. A vocabulary of ~100,000 subword tokens compresses a lexicon of ~600,000 words. Those tokens flow through hundreds of billions of parameters. At each generation step, all 100,000 token types receive a probability score — and one is chosen. That single choice is appended to the input and the whole process repeats.

What feels like "a model writing a sentence" is, at bottom, a sequence of 400+ independent probability-weighted dice rolls — each informed by everything that came before, each backed by 100 trillion arithmetic operations, each producing exactly one next token.

The compression is real, the computation is immense, and the language that emerges is — against all odds — fluent, coherent, and often surprising.
